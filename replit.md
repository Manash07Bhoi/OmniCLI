# OmniCLI

One CLI for file operations, search, conversion, archiving, and workspace management — with a live web dashboard. Built for Termux, Kali Linux, ParrotOS, and Replit.

## Services

Three services run via pnpm workspaces:

| Service | Command | URL |
|---------|---------|-----|
| **Web Dashboard** | `pnpm --filter @workspace/omni-dashboard run dev` | `/` (preview pane) |
| **API Server** | `pnpm --filter @workspace/api-server run dev` | `/api` |
| **Mockup Sandbox** | `pnpm --filter @workspace/mockup-sandbox run dev` | `/__mockup` |

```bash
# Install all dependencies (run once after cloning)
pnpm install

# All three workflows start automatically in Replit.
# To restart a single service manually:
pnpm --filter @workspace/api-server run dev
pnpm --filter @workspace/omni-dashboard run dev

# Check API health
curl http://localhost:8080/api/healthz
# → {"status":"ok"}
```

## Rust CLI

```bash
# Build debug binary (fast; used during development)
cd omnicli && cargo build

# Build optimised release binary (~12 MB, statically linked SQLite)
cd omnicli && cargo build --release

# Run the binary
./omnicli/target/debug/omni --help
./omnicli/target/release/omni --help

# Quick smoke-tests
./omnicli/target/debug/omni file find --path . --type f --long
./omnicli/target/debug/omni convert list
./omnicli/target/debug/omni config show

# Run all tests
cd omnicli && cargo test

# Lint (must stay zero-warning)
cd omnicli && cargo clippy -- -D warnings

# Format
cd omnicli && cargo fmt
```

## Stack

### Rust CLI (`omnicli/`)
- **Language:** Rust 1.88 (edition 2021), workspace with shared dependencies
- **CLI framework:** clap 4 (derive API)
- **Serialisation:** serde, serde_json, toml 0.8, serde_yaml
- **Error handling:** anyhow (CLI boundary), thiserror (per-module typed errors)
- **Hashing:** blake3, sha2, md-5, hex
- **Database:** rusqlite 0.32 (bundled SQLite with FTS5)
- **Archiving:** flate2 (gzip), tar, zip, xz2, bzip2
- **Format conversion:** csv, pulldown-cmark, image, lopdf
- **Encryption:** age 0.10 (X25519 asymmetric)
- **Filesystem:** walkdir, notify
- **Output:** console, indicatif (progress bars)

### API Server (`artifacts/api-server/`)
- **Runtime:** Node.js (v20), TypeScript
- **Framework:** Express 4
- **Database:** Drizzle ORM + SQLite (`lib/db/`)
- **Logging:** pino + pino-http
- **Schema validation:** Zod (generated via Orval from `lib/api-spec/openapi.yaml`)
- **Build:** esbuild (via `build.mjs`), outputs to `dist/`

### Web Dashboard (`artifacts/omni-dashboard/`)
- **Framework:** React 19 + Vite 7
- **Styling:** Tailwind CSS 4 + Radix UI primitives
- **Data fetching:** TanStack Query v5
- **Routing:** Wouter
- **API hooks:** Generated by Orval from OpenAPI spec

## Architecture Decisions

- **No-mock-data rule:** every function returns data computed from real sources. Unimplemented features return `Err(NotYetImplemented)`, never a fake success.
- **thiserror per module, anyhow at the boundary:** each crate defines a typed `ModuleError` enum; `dispatch.rs` converts to `anyhow::Error` for display.
- **BLAKE3 as default hash:** used for all content-hash comparisons (duplicate detection, sync, backup). SHA256 and MD5 are also supported via `--algo` flag.
- **Codec registry is live-generated:** `convert --list` reflects exactly the implemented converters — the list cannot drift from the code (see `list_supported_pairs()` in omni-convert).
- **SQLite FTS5 for search:** index lives at `~/.local/share/omni/search.db`, uses Porter stemmer + unicode61 tokenizer. Path search falls back to LIKE for files outside the FTS index.
- **Dependency rule:** omni-core has zero module dependencies; other crates may depend on omni-core only. omni-file depends on omni-archive (compress delegates to it). The CLI layer (omni-cli) is the only place that imports all modules.
- **API server PORT:** reads `PORT` from env (set automatically by Replit). Hard-coding a port is forbidden — it causes workflow collisions.

## User Preferences

- Zero mock data: no function may return a hardcoded "success" that doesn't reflect a real operation
- Phase-gated development: Phase 3 crates are not created until Phase 2 is verified

## Gotchas

- `cargo build` (not `cargo build --workspace`) from inside `omnicli/` — the workspace root is `omnicli/`, not the repo root
- `libc` crate is needed in omni-core for the `isatty()` TTY probe
- `tempfile` must be listed in each crate's `[dependencies]` (not just `[dev-dependencies]`) because tests are run with the crate enabled by default in Cargo workspaces
- The search index is stored at `~/.local/share/omni/search.db` — run `omni search index <path>` before querying
- `Encryptor::with_recipients` in age 0.10 returns `Option<Encryptor>`, not `Result`
- API server reads `PORT` from env — Replit assigns it; never hard-code a port in the Vite or Express config
- The dashboard makes requests to `/api/*` (relative, no localhost) — this works both on Replit and locally because Vite proxies `/api` to the API server
