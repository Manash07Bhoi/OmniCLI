**Reads with:** `PRD.md` (what/why) + `AGENT.md` (how to build). This file closes the ambiguity a plain PRD leaves open — every gap below gets a concrete decision, not just a flagged question, so Jules never has to guess and silently fill a gap with mock-flavored logic.

Phase 2–4 gaps are logged in §5 as open questions, deliberately *not* resolved yet — answering them now, before Phase 1 ships and teaches us anything, would repeat the over-specification mistake the phased roadmap exists to avoid.

## 1. Blocking Gap — Resolve Before Writing `omni-cli`

PRD §11 says every module (including first-party ones) is "a plugin using the same API," but never defines that API. The dispatcher literally cannot be written without it. Resolved here:

```rust
pub trait OmniModule {
    fn name(&self) -> &'static str;
    fn verbs(&self) -> &'static [&'static str];
    fn execute(&self, verb: &str, args: &ArgMatches, ctx: &OmniContext) -> Result<OmniOutput, OmniError>;
}
```
- `OmniContext` carries: loaded config, DB handles (rusqlite connections, WAL-mode — see §2.4), and resolved styling preference (color on/off, json mode).
- `OmniOutput` is an enum: `Styled(String)` (human output) or `Json(serde_json::Value)` — the dispatcher, not the module, decides which one actually prints, based on `--json`.
- Phase 1 modules are registered in a static compile-time registry (a `Vec<Box<dyn OmniModule>>` built in `main.rs`). Phase 3's WASM/Python loaders implement the *same* trait via a bridge — the interface does not change later, only the loading mechanism gains two more paths.

## 2. Cross-Cutting Conditional Logic & Error Handling

### 2.1 Unified Error Taxonomy
```rust
pub enum OmniError {
    NotFound { path: PathBuf },
    PermissionDenied { path: PathBuf, hint: String },
    InvalidArgument { arg: String, reason: String },
    UnsupportedFormat { from: String, to: String },
    AlreadyExists { path: PathBuf },
    IoError { source: std::io::Error },
    ConfigError { path: PathBuf, reason: String },
    DatabaseError { source: rusqlite::Error },
    NotYetImplemented { module: &'static str, verb: &'static str },
    OperationCancelled,
}
```
Exit codes: `NotFound`/`AlreadyExists`/`PermissionDenied`/`IoError` → `1`; `InvalidArgument`/`UnsupportedFormat`/`ConfigError` → `2`; `OperationCancelled` → `130` (standard SIGINT convention). Every variant's `Display` impl states what failed + the next actionable step (PRD §15) — never a bare debug dump to a non-`--verbose` user.

### 2.2 Global Flag Interaction Matrix
| Combination | Resolved behavior |
|---|---|
| `--json` + `--verbose` | Trace goes to stderr only; stdout stays pure JSON |
| `--json` + `--dry-run` | Plan emitted as `{"dry_run": true, "actions": [...]}`, not styled text |
| `--quiet` + `--verbose` | Rejected at parse time: mutually exclusive, exit `2` |
| `--no-color` + `NO_COLOR` env | OR'd, not overridden — either one disables styling |
| stdout not a TTY (piped) | Color auto-disabled even without `--no-color`, unless `--force-color` |

### 2.3 Destructive-Action Policy (applies to `file clean`, `file copy`/`move` overwrite, `archive extract` overwrite, `backup restore`)
Any operation that deletes or overwrites existing data:
1. Prompts for confirmation interactively, **unless**
2. `--force` is passed → proceeds without prompting, **or**
3. `--dry-run` is passed → shows the plan, asks nothing, changes nothing, **or**
4. stdin is not a TTY (scripted/CI context) **and** `--force` was not passed → **hard fail** with `InvalidArgument`, exit `2`. Never silently proceed and never hang waiting for input that will never arrive — that's the one behavior worse than either alternative.

### 2.4 Concurrency & SQLite
All omni-managed databases (search index, backup manifest, workspace) open with `PRAGMA journal_mode=WAL`. A write that hits `SQLITE_BUSY` retries with bounded backoff (5 attempts over 500ms) before surfacing `OmniError::DatabaseError` — never an infinite retry, never a silent drop.

### 2.5 Signal Handling
`SIGINT` during a long operation (backup, archive, sync) lets the current atomic unit finish (the file currently being copied/hashed), then stops and marks the operation `interrupted` in its status column — distinct from `success`/`failed` — so `backup list`/`verify` reflect real state and `restore` refuses to restore an interrupted snapshot without an explicit `--allow-partial`.

### 2.6 Termux Environment Handling
Detect Termux via `$PREFIX` containing `com.termux`. If a command targets a path under `~/storage/` that doesn't exist yet, emit a specific error pointing at `termux-setup-storage` — never a raw OS "permission denied" with no context.

### 2.7 First-Run / Config Bootstrap
If `~/.config/omni/omni.toml` is absent on first run, `omni` creates it with PRD §10.2 defaults plus `~/.local/share/omni/`, printing a one-line notice **once** (a marker file prevents repeating it). A malformed (present but invalid) `omni.toml` fails fast with the parser's line/column error, exit `2` — never silently falls back to defaults, since that would hide a real mistake the user needs to see.

## 3. Per-Module Gaps

### `omni-core`
- Config precedence, highest to lowest: CLI flag → `OMNI_*` env var → `--config` path or default `omni.toml` → built-in defaults.

### `omni-file`
- `find` pattern: glob by default (gitignore-style via `globset`); `--regex` switches to full regex.
- `copy`/`move` onto an existing destination: fails (`AlreadyExists`) without `--force` (§2.3).
- `duplicate`: symlinks are reported separately, tagged `[symlink]`, never followed for hashing — avoids infinite loops on circular links.
- `clean`'s trash: moves to `~/.local/share/omni/trash/<timestamp>-<name>`, auto-purged after 30 days (configurable); `--permanent` skips trash but still requires `--force`/confirmation.
- `encrypt`: if no `age` keypair exists at `~/.config/omni/age.key`, first use auto-generates one and prints the public key once with a "back this up" notice — never generated silently.
- `sync --watch`: OS-native events (`notify` crate), not polling — battery and correctness both matter on a phone. Conflict (both sides changed) → newer mtime wins, loser saved to trash, never silently discarded.

### `omni-search`
- **Correction to PRD §10.2:** `index_on_idle = true` implies a resident background indexer, which is a process-model commitment not made for Phase 1. Phase 1 indexing is **on-demand only** — `omni search index` or a stale-check scoped to the queried paths at query time. `index_on_idle` is inert until Phase 2; Phase 1 should ignore it with a one-time warning if set `true`, not silently no-op.
- PDF/text extraction: pure-Rust crate (avoids a poppler/C dependency that complicates Termux cross-compilation). Office-doc formats deferred to a Phase-2 `omni install document-formats` package.
- Staleness: mtime+size checked against the index only for files actually matched by a query (cheap); stale hits are labeled `(index may be outdated — run omni search index)`, never served as if fresh.
- Binary detection: null-byte-in-first-8KB heuristic; binaries are indexed by filename/metadata only, content is never extracted.

### `omni-archive`
- **Zip-slip protection is mandatory, not optional:** every extracted entry path is canonicalized and verified to stay inside the target directory; any entry that would escape it aborts the *entire* extraction with a security error rather than partially extracting.
- Password-protected archives: Phase 1 supports *reading* them via `--password` or `OMNI_ARCHIVE_PASSWORD` env var (never a bare CLI arg, to keep it out of shell history/process listings); *creating* password-protected archives is Phase 2.
- Extraction/creation streams in chunks — never loads a whole archive into memory (protects the 80MB RSS target on large files).
- Extracting into a non-empty directory follows the same overwrite policy as `file copy` (§2.3).

### `omni-convert`
- Unsupported pair → `UnsupportedFormat`, exit `2`, message lists the actual live-queried supported matrix (PRD §7.4's no-drift requirement applies to error text too).
- All text conversions assume UTF-8; non-UTF-8 input is a clear encoding error, never silently mangled with lossy replacement characters.
- Lossy formats (e.g. png→webp) default to lossless where the target supports it; `--quality N` opts into lossy explicitly — nothing loses data the user didn't ask to lose.

### `scripts/install.sh`
- Idempotent — re-running detects an existing install and offers an upgrade path instead of erroring.
- Platform detection: `$PREFIX` → Termux (`$PREFIX/bin`); `/etc/os-release` → Kali/ParrotOS/Debian-generic (`~/.local/bin`, no root required, consistent with PRD's no-credit-card/no-root posture).
- Downloaded binary is checksum-verified against the published release manifest (PRD §18) before install; mismatch aborts.
- `install.sh --uninstall` removes the binary only — config/data directories are left in place and reported, since they're user data, not disposable.

## 5. Explicitly Deferred (Phase 2–4) — Logged, Not Resolved

- **Backup algorithm:** full content-hash tree vs. rsync-style rolling delta — decide at Phase 2 start, informed by Phase 1's real hashing throughput, not guessed now.
- **Plugin permission granularity** (per-path vs. per-operation): decide at Phase 3 start alongside the actual `wasmtime` integration.
- **Lua API surface** exposed to user scripts: *(this bullet was self-contradictory — it told Phase 2 to design against Phase-1 workspace usage, but `omni-workspace` is itself a Phase 2 module. Corrected: Lua is now scoped to Phase 3 — PRD §2.2 — decided once Phase 2's real `omni-workspace` usage exists to design against.)*
- **Android companion scope:** Phase 4, optional per PRD §9 — not worth speculating on now.

Resolving these now would mean designing against imagined usage instead of Phase 1's real results — the same mistake the phase gate exists to prevent.
